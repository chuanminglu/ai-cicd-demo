# 提示词06：AI生成边界测试

## 🎭 角色
资深测试工程师，擅长边界值分析和异常场景测试。

## 📋 任务
为修复后的价格计算模块生成完整的边界测试用例。

## 🎯 目标
补充遗漏的边界场景测试，提升测试质量。

## 📤 输出要求

### 测试场景
1. ✅ 正常场景（已有，保留）
2. 🆕 边界场景（优惠等于价格）
3. 🆕 异常场景（优惠超过价格）
4. 🆕 参数校验（负数输入）
5. 🆕 日志验证（优惠超价时记录日志）

### 质量要求
- 使用pytest框架
- 每个测试独立可运行
- 包含断言验证
- 覆盖所有修复点

---

## 💬 复制以下内容到AI

```
你是测试专家。为修复后的代码生成完整的边界测试用例：

【粘贴修复后的 price_calculator.py 代码】

生成测试用例：
1. 保留原有的正常场景测试
2. 新增边界测试：
   - 优惠恰好等于商品价格（应返回0）
   - 优惠超过商品价格（应返回0而不是负数）
   - 优惠远超商品价格（极端情况）
3. 新增参数校验测试：
   - 负数原价（应抛ValueError）
   - 负数优惠券（应抛ValueError）
   - 负数积分（应抛ValueError）
4. 新增日志验证测试：
   - 优惠超价时是否记录WARNING日志

使用pytest框架，直接输出测试代码，不要解释。
```

---

## ✅ 验证输出

生成的测试应该：
- [ ] 包含原有的正常场景测试
- [ ] 包含边界场景测试（优惠超价）
- [ ] 包含参数校验测试（负数输入）
- [ ] 包含日志验证测试
- [ ] 运行 `pytest` 全部通过
